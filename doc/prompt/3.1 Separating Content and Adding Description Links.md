# Separating Content and Adding Description Links

This plan details the changes required to extract discussions and resources data into separate JSON files (`lecture.json`, `resource.json`), support optional clickable links after item descriptions across the SPA, group lectures under course accordion dropdowns, and link resources to specific course/lecture IDs.

## Proposed Changes

---

### [Component: Data Separations & JSON Updates]

We will extract specific sections from `data.json` to new, dedicated files, keeping `data.json` focused on general app structure. We will also add links where appropriate.

#### [MODIFY] [data.json](Documents/GitHub/[oU1TS]/course/data.json)

- Add new `links` array after `description` in `roadmap`, `about.features`, and `join.instructions` blocks.
  - **Roadmap Step 1**: Add link to `#discussions`.
  - **Roadmap Step 2**: Add link to `https://github.com/oU1TS/uits-exam-repo`.

### [Component: UI and Styling Updates]

#### [MODIFY] [style.css](Documents/GitHub/[oU1TS]/course/style.css)
- **Modal Link styles**: Style `.modal-links` to render links nicely aligned inside the modal wrapper.
- **Inline Card Link styles**: Style `.card-links` and `.card-link` to render inline/card-level links for About and Join sections.
- **Accordion Dropdown styles**: Style `.courses-accordion`, `.course-accordion-item`, `.course-accordion-header`, and `.course-accordion-content` with CSS transitions (expanding/collapsing using max-height).
- **Redirection Link styles**: Style the related lecture buttons on resource cards to align and transition nicely on hover.
- **Visual Highlight**: Add a `.highlight-flash` animation that briefly pulses a border/background color to draw the user's attention to a redirected item.

### [Component: Client SPA Controller]

#### [MODIFY] [app.js](Documents/GitHub/[oU1TS]/course/app.js)
- **Data Caching & Loading Logic**:
  - Implement a multi-file state cache:
    ```javascript
    let appState = null;      // data.json
    let lecturesState = null;  // lecture.json
    let resourcesState = null; // resource.json
    ```
  - Rewrite `fetchAppData(fileName)` to fetch and store whichever JSON file is requested based on its filename.
- **Routing Engine**:
  - Update `routePage()` to parse query parameters (e.g. `#discussions?lecture=cse-211-oop-l1` or `#discussions?course=cse-213-dsa`).
  - Pass the parsed `queryParams` to `setupViewInteractions(renderKey, queryParams)`.
  - Update `routePage()` to load the appropriate file corresponding to the active hash segment:
    - `#home`, `#about`, `#join` -> fetch `data.json`.
    - `#discussions` -> fetch `lecture.json`.
    - `#resources` -> fetch `resource.json`.
- **Render Functions**:
  - Update `Render.discussions()` to render the courses as collapsible accordion dropdowns.
  - Update `Render.resources()` to detect `relatedLectureId` and `relatedCourseId` and add a button navigating to `#discussions?lecture=...` or `#discussions?course=...`.
  - Update `showRoadmapModal(stepIndex)` to check for step links and render them inside the `#modal-links` container. Note: If the link is an internal SPA hash (e.g. `#discussions`), clicking it will automatically close the modal.
  - Update `Render.about()` to dynamically render links after descriptions in the "features" section if a `links` array is defined.
  - Update `Render.join()` to dynamically render links after descriptions in the "instructions" timeline if a `links` array is defined.
- **View Interactions**:
  - In `setupViewInteractions()`, handle accordion toggle click event listeners.
  - If a `lecture` or `course` query parameter is present, automatically trigger the accordion header's expand action, scroll the target element into view smoothly, and apply the `.highlight-flash` class.
